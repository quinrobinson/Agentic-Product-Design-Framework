# CLAUDE.md — Agentic-Product-Design-Framework

## Project Overview

An AI-integrated UX design framework built around a 6-phase workflow: **Discover → Define → Ideate → Prototype → Validate → Deliver**. Deployed as a live GitHub Pages site with interactive tools, plus a Node.js MCP server for Claude Code integration.

**Live site:** [quinrobinson.github.io/agentic-product-design-framework](https://quinrobinson.github.io/agentic-product-design-framework/)

---

## Commands

```bash
# Web app
cd web && npm install
npm run dev          # dev server (Vite, localhost:5173)
npm run build        # production build to web/dist/
npm run lint         # ESLint

# MCP server
cd mcp && npm install
npm run build        # tsc → mcp/dist/index.js
npm run dev          # tsc --watch
```

**Verify MCP server:**
```bash
echo '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}' | node mcp/dist/index.js
```

## Architecture

### Web (`web/src/`)
Vite + React 19 SPA. Each `.jsx` file in `src/` is a standalone tool component (e.g. `ResearchSynthesizer.jsx`, `ProblemFramingTool.jsx`). `App.jsx` is the router/shell.

**Important:** `SkillsLibrary.jsx` is a standalone deployable overlay — it is **not** imported by `App.jsx`. If skill data is updated, sync both `App.jsx`'s inline `SkillsLibraryOverlay` and `SkillsLibrary.jsx`.

### MCP server (`mcp/src/`)
Node.js stdio MCP server with 18 tools. Each tool returns a structured prompt that Claude processes — no external API calls.

### Skills (`skills/`)
Markdown skill files for each phase. Upload the relevant `.md` to a Claude conversation to activate phase-specific workflows and quality checklists.

### Artifacts (`artifacts/`)
JSX interactive tools rendered via the live GitHub Pages site.

## Project Context & Hooks

- **`.apdf/context.json`** — Fill in before using MCP tools. Copy from `.apdf/context.json.example`. Required fields: `project_name`, `phase`, `persona`, `problem_statement`, `constraints`, `open_questions`, `last_handoff`.
- **`.claude/settings.json`** — Pre-configured hooks:
  - `PreToolUse` → `inject-context.sh` — injects `.apdf/context.json` before every `mcp__apdf__*` call
  - `PostToolUse` → `persist-artifact.sh` — saves tool output to `.apdf/artifacts/`
  - `Stop` → `phase-routing.sh` — fires at session end

All APDF MCP tools are namespaced `mcp__apdf__<tool_name>`.
